한국 골프존 골프장 음식점에서 수집된 6가지 핵심 피드백

🔴 피드백 #1 — 골프존 ERP 연동 필수

골프존이 골프장에 자체 ERP를 공급하고 있으며, 부분적으로 스펙이 상이함. OQ가 골프존 ERP와 반드시 연동되어야 함.

🔴 피드백 #2 — 티오프 타임 단위 주문

음식 주문은 티오프 타임 단위로 관리. 유저 인식 단위 = "티오프 타임의 누구". 개인이 아닌 그룹 기반.

🟡 피드백 #3 — 주문 방식 미정

태블릿으로 할 수도 있고, QR을 선택해도 됨. 아직 미정 상태.

🔴 피드백 #4 — ERP 메뉴 객체 동기화

골프존 ERP에 메뉴판 객체가 등록되어야 함. 동기화 방식으로 연동.

🔴 피드백 #5 — 3부(야간) 선결제 예외

1부(오전)·2부(오후)는 후결제이나, 3부(야간)는 선결제를 한다고 함. 선/후결제 혼합 환경.

🔴 피드백 #6 — 결제 단말 다양성

앱카드 없는 유저, 캣단말기만 있는 시골 골프장 등 결제 인프라가 다양함. 카드 결제 단말(모바일 포함) 연결 가능 여부 문의. "POS와 싸우기 싫어서 안 했다"고 답변한 상태.

1 핵심 아키텍처: 골프존 ERP 연동

골프존 ERP를 중심으로 OQ가 연동되는 구조

💡 핵심 원칙

ERP가 마스터 데이터를 관리하고, OQ는 주문 경험과 AI 부가가치를 담당합니다.

2 솔루션 #1: 티오프 타임 기반 주문 시스템

유저 인식 단위 = 티오프 타임 → 팀(조) → 개인

🎯 주문 인식 체계

레벨식별자예시용도
L1. 티오프 타임날짜 + 시간2026-03-09 10:30최상위 그룹
L2. 팀(조)티오프 내 조 번호A조 (4명)주문·정산 단위
L3. 개인팀 내 개인 번호A조 2번 홍길동개별 메뉴 선택

📱 QR / 태블릿 주문 플로우

STEP 1 — 인증
고객이 QR 스캔 or 태블릿에서 티오프 타임 선택
골프존 ERP에서 해당 티오프 타임의 팀 목록을 가져옴. 고객이 자신의 팀(조) 선택.
STEP 2 — 메뉴 확인
AI 디지털 메뉴판 표시
ERP에서 동기화된 메뉴 + OQ AI가 생성한 이미지·설명. 외국인 골퍼를 위한 다국어 지원.
STEP 3 — 주문
팀 단위 "함께 담기" → 주문 확정
같은 티오프 타임 팀원들이 각자 메뉴를 담고, 한꺼번에 주문 확정. ERP에 주문 데이터 전송.
STEP 4 — 결제
후결제(1·2부) or 선결제(3부) 분기
결제 방식에 따라 분기 처리. 아래 솔루션 #2에서 상세 설명.
📋 QR 모드
  • 테이블/좌석에 QR 부착
  • 스캔 → 티오프 타임 자동 매칭
  • 고객 개인 스마트폰에서 주문
  • 장점: 하드웨어 투자 불필요
📱 태블릿 모드
  • 테이블에 태블릿 비치
  • 터치로 티오프 타임 선택
  • 공용 태블릿에서 그룹 주문
  • 장점: 사용 편의성, 시니어 대응
제안: 골프장 특성상 시니어 이용객이 많으므로, 태블릿 모드를 기본으로 하되 QR 모드를 보조 옵션으로 제공하는 것을 권장합니다.

3 솔루션 #2: 선결제/후결제 혼합 결제 체계

1·2부 후결제 + 3부 선결제 + 다양한 결제 인프라 대응

⏰ 부별 결제 방식

구분시간대결제 방식결제 시점OQ 처리
1부오전후결제라운드 종료 후주문 누적 → 일괄 정산
2부오후후결제라운드 종료 후주문 누적 → 일괄 정산
3부야간선결제주문 시 즉시주문 시 결제 → 즉시 확정

💳 결제 인프라별 대응 방안

방안 A: 앱카드 결제 (도심 골프장)

앱카드 보유 고객 → OQ 내 앱카드 간편결제 연동. 삼성페이, 카카오페이 등 간편결제 SDK 통합.

장점 빠른 결제, POS 미경유 적합 도심·신규 골프장

방안 B: VAN 직접 연동 (OQ 강점 활용)

OQ의 기존 VAN 직접 연동 기술을 골프장에 적용. PG 수수료 없이 VAN사 직접 연동, 당일 입금.

장점 PG 수수료 제거, 당일 입금 주의 기존 POS와 충돌 가능성

방안 C: 기존 CAT 단말기 활용 (시골 골프장)

캣단말기만 있는 시골 골프장 → OQ에서 주문만 받고, 결제는 기존 CAT 단말기에서 처리. 주문-결제 분리 모델.

  • OQ에서 주문 확정 → 주문서(금액) 생성
  • 직원이 CAT 단말기에서 해당 금액 결제 처리
  • 결제 완료 후 OQ에서 수동 확인 or ERP 연동으로 자동 확인

장점 추가 하드웨어 투자 불필요 적합 시골·소규모 골프장

방안 D: 모바일 결제 단말 연동 (하이브리드)

블루투스 모바일 카드 리더기(예: 페이히어, 한국정보통신 모바일 단말) 연동. OQ 앱에서 주문 후 모바일 단말로 결제.

  • OQ 주문 확정 → 모바일 단말에 결제 금액 자동 전송
  • 고객이 카드를 모바일 단말에 터치/삽입
  • 결제 완료 → OQ에 자동 반영

장점 POS 불필요, 이동식 결제 가능 혁신 서빙 직원이 테이블에서 바로 결제

🎯 "POS와 안 싸우는" 전략

기존 입장: "포스랑 싸우기 싫어서 안 했다"
새로운 접근: POS와 싸우지 않고, POS가 없는 영역을 공략한다.
❌ POS와 충돌하는 영역
  • 기존 POS 결제 대체
  • 매출 데이터 가로채기
  • POS 하드웨어 교체 요구
✅ POS 공백 영역 (OQ 기회)
  • 디지털 메뉴판 (POS에 없는 기능)
  • 다국어 지원 (POS에 없는 기능)
  • AI 이미지/설명 (POS에 없는 기능)
  • QR 기반 셀프 주문 접수
  • 3부(야간) 선결제 모바일 결제
  • 시골 골프장 CAT 연동 주문

4 솔루션 #3: 골프장 유형별 맞춤 패키지

도심 대형 골프장 vs 시골 소규모 골프장에 맞는 차별화된 솔루션

🏙️ Type A: 도심 대형 골프장

특징

  • 골프존 ERP 최신 버전
  • 앱카드·간편결제 가능 고객 다수
  • 외국인 골퍼 방문 빈도 높음
  • IT 인프라 양호

OQ 패키지

  • ✅ 태블릿 + QR 듀얼 모드
  • ✅ 골프존 ERP 풀 연동
  • ✅ 앱카드 간편결제
  • ✅ 다국어 AI 메뉴판
  • ✅ CRM 알림톡 연동

월 요금: 49,900원/월

🌾 Type B: 시골 소규모 골프장

특징

  • 골프존 ERP 구버전 or 없음
  • CAT 단말기만 보유
  • 내국인 중심
  • IT 인프라 취약

OQ 패키지

  • ✅ QR 모드 (태블릿 불필요)
  • ✅ ERP 라이트 연동 or 독립 운영
  • ✅ CAT 단말기 연계 (주문-결제 분리)
  • ✅ AI 메뉴판 (한국어)
  • ✅ 직원용 주문 관리 웹앱

월 요금: 19,900원/월

5 솔루션 #4: 3부(야간) 선결제 특화

야간 영업의 선결제 요구사항을 OQ VAN 직결로 해결

💡 OQ의 킬러 기능: VAN 직접 연동

OQ는 이미 PG 수수료 없이 VAN사 직접 연동, 당일 입금이라는 업계 유일 기능을 보유하고 있습니다. 이것을 골프장 3부 선결제에 적용하면, POS와 싸우지 않으면서도 선결제 영역을 확보할 수 있습니다.

🌙 3부 선결제 플로우

STEP 1
고객이 QR/태블릿에서 메뉴 선택
3부 시간대 자동 감지 → 선결제 모드 활성화
STEP 2
OQ VAN 직결 결제
앱카드 or 카드번호 입력 → VAN사 직접 승인. PG 미경유, 수수료 절감.
STEP 3
결제 완료 → 주문 확정 → 주방 전송
결제 완료 시점에 주문이 확정되어 주방 KDS로 전송. ERP에도 동기화.
STEP 4
당일 입금
VAN 직결이므로 매출이 당일 입금. 골프장 현금 흐름 개선.

🎯 3부 선결제가 OQ의 "포스 안 싸우기" 전략의 핵심

1·2부는 기존 POS/CAT으로 후결제 유지 → POS와 충돌 없음
3부만 OQ VAN 직결 선결제 → 새로운 매출 영역 개척
골프장 입장에서는 "3부 선결제 문제를 해결해주는 솔루션"으로 OQ를 인식

7 개발 로드맵

단계별 구현 계획

Phase 1 (1~2개월)
골프존 ERP 연동 PoC
ERP API 스펙 분석, 메뉴 객체 동기화 프로토타입, 1개 골프장 파일럿. QR 메뉴판 MVP.
Phase 2 (3~4개월)
티오프 타임 기반 주문 시스템
티오프 타임 인증, 팀 단위 함께 담기, 후결제 정산 연동. 태블릿 모드 개발.
Phase 3 (5~6개월)
3부 선결제 + VAN 직결
야간 선결제 모드, VAN 직결 결제, 앱카드 간편결제 연동.
Phase 4 (7개월~)
시골 골프장 확장 + 모바일 단말
CAT 연동 모드, 모바일 결제 단말 연동, 전국 골프장 롤아웃.
