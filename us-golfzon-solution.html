미국 골프존 현장에서 수집된 4가지 핵심 피드백

🔴 피드백 #1 — TOAST POS 종속

미국 레스토랑은 무조건 TOAST를 쓴다. TOAST KDS가 매우 튼튼하고, 알바생들도 TOAST에 익숙해져 있음. POS 교체 의지 없음.

🔴 피드백 #2 — QR/태블릿 오더 부적합

미국 스크린골프장은 아케이드장/베가스 호텔 로비 같은 분위기. 방 구조가 아니라 오픈 공간. 서버가 직접 서빙. QR 오더·태블릿 오더가 맞지 않음.

🟡 피드백 #3 — 메뉴판/다국어 니즈 존재

서버가 주문을 받더라도 국제화된 메뉴판은 필요할 수 있음. "함께 담기" 같은 기능으로 서버 주문을 더 우아하게 받는 시나리오 가능.

🟡 피드백 #4 — 라커 단위 결제

유저 인식을 라커 단위로 결제한다고 함. 개인이 아닌 라커 그룹 단위 정산 체계.

1 핵심 진단: OQ의 포지션 재정의

TOAST를 적으로 만들지 말고, TOAST 생태계 위에서 부가가치를 만드는 파트너가 되어야 합니다.

2 솔루션 #1: TOAST 연동 전략

TOAST API를 활용한 메뉴 동기화 + AI 부가가치

🔧 TOAST Open API 활용

TOAST는 Open API(Restaurant API)를 제공합니다. OQ가 TOAST의 메뉴 데이터를 실시간 동기화하면, 매장 운영자가 TOAST에서만 메뉴를 관리하면서도 OQ의 AI 기능을 활용할 수 있습니다.

3 솔루션 #2: 서버 어시스트 모드

QR 오더가 아닌, 서버의 주문 프로세스를 강화하는 방식

핵심 컨셉: "QR로 주문하세요"가 아니라 "QR로 메뉴를 보세요, 서버가 주문 받아드립니다"

🎯 서버 어시스트 모드 시나리오

STEP 1
고객이 QR 스캔 → 다국어 디지털 메뉴 확인
주문 버튼 없음. 메뉴 열람 전용. 11개 언어 지원으로 외국인 고객도 쉽게 메뉴 파악.
STEP 2
고객이 "함께 담기" 장바구니에 원하는 메뉴 담기
직접 결제가 아님. "나 이거 먹고 싶어" 리스트를 만드는 것. 라커 그룹 내 여러 명이 각자 담기 가능.
STEP 3
서버가 라커 번호로 장바구니 확인 → 주문 확정
서버 전용 앱에서 해당 라커의 장바구니를 확인하고, 고객과 대화 후 주문 확정. TOAST KDS로 전송.
STEP 4
서버가 음식 서빙
기존 서빙 프로세스 유지. 달라지는 것: 주문 정확도 향상 + 언어 장벽 해소.

📱 서버 전용 앱 기능

🔍 라커별 장바구니 대시보드

각 라커 그룹이 담은 메뉴를 실시간으로 확인. 서버가 테이블에 가기 전에 미리 주문 내용 파악.

🌍 자동 번역 커뮤니케이션

고객이 한국어로 담은 메뉴를 서버에게는 영어로 표시. 역방향도 지원.

⚡ 원터치 TOAST 전송

장바구니 확인 후 "주문 확정" 버튼 한 번으로 TOAST KDS에 자동 전송.

📊 라커별 누적 주문 현황

라커 단위 결제에 맞춰 누적 주문·금액 추적. 정산 시 참고용.

4 솔루션 #3: 라커 기반 그룹 오더

라커 번호 = 유저 그룹 ID, 결제 단위

🔑 라커 기반 인증 플로우

고객이 라커 번호를 받으면 → 해당 라커 QR코드 스캔 → 자동으로 해당 그룹에 합류 → 그룹 내 모든 구성원이 메뉴를 "함께 담기" → 서버가 확인 후 주문 확정

기능설명기존 대비 장점
라커 QR라커마다 고유 QR 부착. 스캔 시 자동 그룹 합류별도 인증 불필요
함께 담기그룹 내 여러 명이 동시에 장바구니에 메뉴 추가서버에게 구두 전달 오류 방지
다국어 메뉴각자 모국어로 메뉴 확인 가능한국인·미국인 혼합 그룹 대응
그룹 정산라커 단위 누적 주문 금액 추적TOAST 정산과 연동

💡 킬러 유스케이스: 한국인 골퍼 그룹

미국 골프존에 한국인 골퍼 4명이 방문 → 라커 QR 스캔 → 한국어 메뉴판으로 각자 먹고 싶은 것을 담기 → 서버가 영어로 확인 후 TOAST에 주문 → 언어 장벽 제로, 주문 실수 제로
